## 1. Acceptance Criteria (measurable)

1. **Progress field exists** вҖ“ `Recommendation` model must expose a `progress` attribute of type `float` (0.0вҖ“1.0) with default `0.0`.  
2. **Progress bar rendering** вҖ“ Dashboard component renders a `<progress>` element whose `value` equals `recommendation.progress` and `max="1"`.  
3. **Automatic update** вҖ“ When a `Course` instance is marked `completed=True`, the related `Recommendation.progress` updates to reflect the new completion percentage (e.g., 0.5 if 2 of 4 courses are complete).  
4. **Modal detail** вҖ“ Clicking the вҖңView ProgressвҖқ button opens a modal that lists each course with its completion status and shows the overall percentage.  
5. **Persisted state** вҖ“ Progress updates survive a full page reload (i.e., stored in the database).  
6. **Edge case вҖ“ no courses** вҖ“ A recommendation with zero courses shows a progress bar at `0%` and a message вҖңNo courses assignedвҖқ.  
7. **Security** вҖ“ No sensitive data (e.g., user IDs) is exposed in the progress payload returned to the client.
